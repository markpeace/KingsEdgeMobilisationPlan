# North Star

## What we are producing

We are producing an interactive mobilisation site for the King's Edge programme.

The site should work as both a strategic map and a practical mobilisation tool. It should be clear enough for senior discussion, detailed enough to support delivery planning, and structured enough to evolve as the programme matures.

## Purpose of the site

The site will help users understand how King's Edge turns the graduate premium into a deliverable programme of work.

It will show how King's Edge connects:

- curriculum-embedded graduate value
- co-curricular opportunity and purpose
- extra-curricular belonging, navigation and student life
- evidence and articulation of the graduate premium

## Core hierarchy

The site should render the plan through this hierarchy:

**King's Edge -> Projects -> Deliverables -> Components -> Delivery steps**

At the top level, users should see four projects:

1. Curriculum-Embedded Graduate Premium
2. Co-Curricular Opportunity and Purpose
3. Extra-Curricular Student Life, Navigation and Belonging
4. Evidence and Articulation of the King's Graduate Premium

Each project contains four deliverables. Each deliverable contains a summary, owner, components, delivery steps, dependencies and related deliverables.

## What the site needs to make easy

Users should be able to:

- browse by project
- browse by deliverable
- click into a detailed deliverable page
- understand who owns each area
- see what problem each deliverable solves
- see what changes as a consequence of the work
- view the plan as a timeline / Gantt
- understand dependencies between deliverables
- see the three major cross-programme dependencies

## Design principle

The plan should not read as a static document. It should behave like a browsable delivery map.

A user should be able to move from the whole programme view to a specific deliverable, then back out to the timeline or dependency view.
